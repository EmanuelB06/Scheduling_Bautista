List the workflow of using Git.

1. Open cmd (Windows + R)
2. Initialize your local repository (git init)
3. git remote add origin "your repository link"
4. git add .
5. git commit -m "description"
6. git branch -M main
7. git push -u origin main
=======
1. Commit often: Small, frequent commits make tracking changes easier. They also simplify rolling back code if errors surface.
2. Never commit broken code: Run your project local build tools and tests successfully before staging. Ensure your changes do not break existing functionality.
3.Use the staging area: Leverage git add -p to review and selectively stage precise lines of code. Avoid blindly running git add . on a messy directory.
